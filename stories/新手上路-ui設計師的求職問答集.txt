這幾年由於網路、科技的蓬勃發展，UI設計師這個職位真的越來越熱門了，各種不同類型的產業(甚至傳產要轉型)都需要相關人才的投入，但偏偏目前學校傳授這門領域的科系並不多，在科技快速迭代與變化的今天，學校所學的知識與觀念常常和業界的的需求有所落差，造成遇過許多剛畢業或是想轉行的朋友對於真實的產業運作一知半解，也不清楚該如何開始，因此今天整理了幾個常被問到、關於UI設計師求職的的問題，希望可以幫助一些想踏入UI設計領域的朋友解惑，以下內容大多為個人經驗與觀點，如有不同的想法也請分享賜教、彼此交流:)Hello 我是 Simon，目前是一名數位產品設計師，過去的背景是平面設計，出社會的期間也不務正業待過餐飲業、在國外Working Holiday等等，從去年開始轉職為UI設計師，今年開始試著做自己的產品，同時更深入的學習UX與前端程式。在轉職與學習的路上因為對於這塊領域的不了解，也踩了不少坑，走了許多冤枉路，因此想把一些小小的經驗寫出來和大家分享，希望對正在看這篇文章的你有所幫助。從名稱上來看，網頁設計師(Web Designer)和UI設計師(User Interface Designer)好像看不太出來實際工作內容，網頁設計師一般來說指的是設計產出網頁的視覺呈現，細分還可以分為純做視覺執行，或是視覺兼前端切版(HTML+CSS)，當然，現在響應式網頁(RWD)當道，網頁設計師也要懂的在不同裝置與平台上版面該如何配置與呈現。而UI設計師又稍微複雜一點，除了視覺呈現外同時也要顧及使用體驗，對於流程規劃與使用的細節上又更為重視一點，UI設計師不一定會設計到網頁，也有可能設計App介面，也有可能都要會，端看公司的產品屬性而定。就我的觀察，這兩種職位常常會會出現在不同的產業當中，許多網頁視覺設計師常常需要產出所謂的活動網頁，有許多華麗的動態效果，而UI設計師做的網頁比較偏向功能性與操作性等跟流程比較有關的網頁(如後台介面、大型商務網站等)。坦白說這和產業型態、公司資源與團隊編制有非常直接的關係，一般來說在大型企業或科技公司裡面的分工較為精細，每個角色所負責的工作範圍也較為明確與固定，所以常會看到比較精準的職位需求像是UI設計師、UX設計師、產品經理(PM)等等。但目前以台灣來說，其實還是以中小企業為大宗(尤其許多是接案公司、新創公司)，對於設計端所提供的資源較少，往往需要的是較為全面的人才(意思是什麼都要會一點，UI/UX都要懂最好還會寫點程式最好)，才能適應多變的環境，加上有些雇主並不真的了解UI或UX在職能上的實際差異，所以往往開出的職缺工作內容會比較雜一點，至於要如何選擇，端看你自己在未來想要成為什麼樣的設計師而定，是想要培養多方領域的能力？還是想要在某個技能上鑽研得更深？這必須要去審慎地思考，新創公司和大公司所學習到或經歷到的事情絕對是非常不同的。如果你想找的職位是「UI設計師」，那必須要考慮一下要求會寫程式的公司，因為這很可能會比較接近網頁設計師的職位，會花時間要切版、修點小bug，相對的在設計層面時間也就更少了，也就更少有機會去接觸使用體驗、流程規劃的部分，但如果你想精進程式方面的能力就另當別論。我知道很多公司會把HTML+CSS列為加分的能力條件，有些雇主或是HR也會認為能力需求開得越多越好，但其實這樣找來的人也未必能很專精在單一領域，進而有效的幫產品解決問題。至於「UI設計師要不要會寫程式」這題算是萬年老題，我自己的觀察是：「會很好，不會也沒關係。」當然，會寫程式可以帶來許多好處，一來你可以更清楚你的設計是怎麼被實作的，二來你在設計的過程會用更系統性的思維去做設計(但老實說懂程式有時會限制了自己的創意)。但就算你不會寫程式我覺得也不需要太擔心，因為嚴格說起來寫程式真的不算在UI設計師的職能範圍內(但大概要懂一點程式的概念)，它只是有助你更好的和工程師協作的能力之一而已，身為設計師應該要花更多心力去挖掘產品的核心價值與提供更好的解決方案。以上是開玩笑的，現在UI設計的工具非常的多元，現在幾乎可以說是百花齊放的時期，不管是 Mac 和 Windows 的用戶都能找到適合的工具來做，至於為什麼 Sketch 目前是UI設計的主流？我的觀察是 Sketch 有效的解決了數位產品設計師在工作時的痛點，不管是提升效率或是團隊協作都非常有幫助，並且擁有極為豐富的社群資源以及第三方協作平台在背後支撐，當然還有其他軟體像是AdobeXD、Figma等等，但我認為這幾款工具目前要追上 Sketch 都還需要一段時間，如果你是都還沒有接觸過這類設計工具的朋友，可以都去玩玩看，比較之中的差異再選擇適合自己的工具即可，如果想更了解 Sketch 這套工作如何幫助你提高效率，可以參考我之前寫過的這篇文章：設計的基本觀念像是配色、字體、空間安排原則是相通的，但在App和網頁使用流程與相關設計規範其實大有不同，網頁設計你必須懂RWD，App設計你必須要了解 iOS 和 Android 雙平台的設計規範以及許多 App 常見的操作模式，就連手機版網頁和App的使用習慣上也有許多不同，因此設計師在設計時需要考慮的方向也要做調整，我自己認為如果要往成熟的設計師前進，最好網頁設計和App設計都要接觸過會較好。至於網頁和App能不能用 Sketch 製作？答案當然是可以的(我現在連平面設計都用Sketch做XD)，唯一要注意的是在匯出圖檔時網頁和 App的需求略有不同，不過當你有實作經驗之後這些都不太是問題才是。如果是問我，我會回答：「透過網路自學。」現在已經是一個透過網路你不可能找不到答案的時代，從一個問題的解答可以延伸出成千上萬的知識點，透過這些新的知識點你可以不斷挖掘新的知識，慢慢建構出自己的學習系統。不過自學相對來說學習曲線會比較陡峭、時間也會拉得比較長，走起來也有可能非常顛頗，但我認為失敗也是一種學習，它可以讓你的心態變得更堅強。但如果你是急於就業或想轉職，找老師入門可能是最快的解決方案，你可以快速從有實作經驗的老師身上學到業界要用的技能，相對代價會比較高，坊間有不少在傳授UI/UX相關的課程或補習班，不過可惜的是通常要價不菲，對於新鮮人來說可能是一筆負擔，我認為線上課程可能是另外一種不錯的方案，用較小的金額並且可以重複學習，當你開始探索出一點心得之後再去選擇適合自己的實體課程也不遲，當然 Youtube 也有許多免費的教學影片可以看。講到線上課程，如果你是完全沒有經驗的人，歡迎看看我在 Hahow所開設的 UI設計入門＆Skech 設計線上課程(這不是廣告，試看不用錢XD)可以說是有點重要也不太重要，為什麼呢？如果你今天是剛畢業並且目標是大公司，學歷可能會有點重要，因為大公司們HR們常常第一關篩選的標準可能是學歷與經歷，但如果你想進入的是新創公司，相對的你的熱情與實力會比學歷要來得重要，要如何表現出你對這個產業的興趣以及展現你的個人特質會比你目前能做出什麼東西還重要，許多公司看的是潛力而不是你此刻的實力，當然，你還是需要有這個領域最基本的能力，才有可能拿到面試的入場券。我認為沒經驗其實不是最大的問題，你可以透過很多方法累積經驗，做Side Project、Redesign、做Daily UI Challenge等等都可以幫助你累積作品，當你展現足夠的熱情並且拿出作品，即便不是最成熟的作品，但我相信不至於完全沒有面試的機會，從面試的過程中你也可以了解每間公司需要人才的是什麼，反覆把自己不足的部分再次補強。對我來說，作品集是讓面試的公司最快瞭解自己的管道，所以必須在最快的時間抓住觀者的眼球，盡可能的表現出你的實力與熱情。我相信好的作品會說話，作品中也要盡量表現出你的你是如何思考、為何而做這個設計，設計的背後需要有原因和脈絡的，最重要的是你的設計解決了什麼問題。我認為作品在精不在多，不需要把你所有作品都展現出來，尤其在學生時期所產出的東西很多只能算「作業」而稱不上「作品」，只需要把你最有把握且完整的2–3個作品放上去就好，而作品集的載體非常建議使用個人網頁，個人網頁不僅可以朔造個人的形象，也非常即時、符合時代趨勢，要更新也非常方便，我相信很多企業主甚至會在手機瀏覽你的作品，因此會比紙本或電子書形式的作品集更為有說服力。另外最好可以針對每間要應徵的公司客製化履歷，也要對該公司的產品或服務有一定的了解，把一些優勢強化在你的經歷裡。面試最基本的就是要把自己最好的一面呈現出來，強調自己的優勢以及說明自己為何符合要應聘的職位，在事前也要做好功課，大致了解這間公司的文化、公司的的產品或服務有什麼優缺點、或是他們注重的是什麼，知己知彼，百戰百勝，越了解對方你的應答才能越自然。在表達自己的作品時也要技巧性的避掉一些自己較弱的部分，盡量多講「你為什麼這樣做」，設計背後的原因很重要，通常面試官在意的是你的設計流程、你是如何思考、是如何找到問題的核心與提出解決方案，對他們來說，美感固然重要，但更重要的是你有沒有真正的解決問題。最後也要提醒各位，不只是公司面試你，你也在面試公司，真的要選一間自己喜歡的公司(很重要)，在面試時也一併觀察公司其他員工的狀態，或是多問點實際工作的問題，不然很有可以進去之後跟想像的有所落差。如第二題有提到的，對於UI設計師這個角色來說，不同產業和不同的團隊編制會很直接影像到你的工作內容，接案公司可以在短時間內接觸到許多不同類型的案件，對於訓練實力很有幫助，但通常會會比較累，而且案子做的不深，無法有太多時間了解整個案子，做出來的東西也比較偏重視覺呈現，1個人要應付好幾個案子是很常見的情況。新創團隊的特色通常就是團隊年輕、有活力，變化也很快，如果是做自己的產品也會有更多時間研究用戶(不過也要看產品是否已經在賺錢，有時間做研究)，大公司則是分工更細，更專注在你的角色技能上。不過每間公司都有不同的文化以及不同的職涯發展性，以上只是概略參考，還是要思考清楚在未來你想要往什麼方向發展，老話一句，關於工作的細節一樣是面試時要問清楚。數位產品設計領域其實是非常非常非常廣的，基本上永遠都有學不完的東西出現，每個領域的 Domain-Knowhow 都相差甚遠，今天你做金融產業、做汽車產業、做電商、做交友等，需要了解的背景知識完全不一樣，更而況是需要透過設計語言傳達訊息給使用者，這肯定是需要花時間去適應的，但這同時也是做UI設計有趣的地方，科技、產業市場急遽快速的變化讓我們必須不斷的吸收新知識，同時適應各種突如其來的狀況。至於要如何成為獨當一面的UI設計師，我會說：這是一條漫漫長路。「心態」我想是最重要的一的部分，你必須要有健康的心態，在面對挑戰時試著找方法解決，失敗是難免的，但從經驗的累積中你一定會有所成長。不恥下問也很重要，一定要很常去問「為什麼？」，其實沒有人會在乎你問了什麼看起來很笨的問題，我想沒有人是全能的，每個人的知識都會有缺口，當我們問得越多，我們就越能組織起自己的學習策略，可以越學越快，也越能掌握世界的變化。另外經驗是需要時間累積的，即使是天才他也需要透過嘗試失敗讓自己成長，因此不斷學習、嘗試錯誤、勇於問問題，我想這會是我建議的的方法。以上就是我的一點心得的分享，雖然這篇文章著力在UI設計的部分，但對我來說UX的思維也非常重要，當你已經熟練地掌握工具或是技術，更應該往回走，去了解產品設計的核心以及使用者或是在乎客戶在意的是什麼，有了實作經驗加上方法論去支持的你設計成果，這樣更能做出打動人的設計。希望這篇文章有幫助到各位，如果有任何其他疑惑或想法也歡迎在下方留言，我會盡我所能回答的:D我是Simon，一名產品設計師，目前正在東京工作。我喜歡分享、試著從小事開始創造價值，我的興趣很多、話也不少，如果有合作&交流機會或任何想聊的事情，都不用客氣透過我的

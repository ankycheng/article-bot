首先～先介紹一下我的背景好了，2016年從台科大主修商設畢業，GPA平均3.8左右，在滿是人才的設計系裡，作品與成績都不算最出色。曾經很茫然不知道念設計可以幹嘛，所以玩很多活動、組團等，還一度還想要歪樓去唱歌（希望父母沒有因此折壽）但總之現在微步入正軌，擁有兩年半的介面設計經驗，希望可以分享心得給學弟妹。我到底想學什麼？我想很多人跟我一樣是到了學測成績出來以後，才第一次面對「我到底想學什麼」這個問題，才漸漸地去了解自己，漸漸地去瞭解科系，接著在四年內從所選的科系範圍中，找到有興趣的、更細的專業，因此困境就來了～如果不去思考「我到底想學什麼」便有很大的機會將自己的專業受限於學校的教課重點，這時候就會出現無法學以致用、未來茫茫然、選錯科系（？）、我是爛草莓啃老族之類情境。學校給我們什麼我們就學什麼？以台科我那屆為例，最大的教學重點也是大多數的畢業製作主題為「包裝與動畫」，像我這種美工刀拿不好，只會把 Maya 搞當機內心媽啊的設計系學生，真的很難想像自己會順利畢業。還好當時藉由文字造型課，接觸到介面設計，儘管離真正的實作還是有很大的距離，但我看到了這扇門，就算不是學校教課重點，但只要服用其他主動學習的方式，都非常有機會跳脫學校開始自學 UI/UX。驅使我快速學習介面設計的原因，是大二暑假開始的創業項目，讓我有直接設計產品的機會，很幸運的還有短暫登過 App Store 工具類下載排行榜第一，但這裡沒有鼓吹學生創業的意思，這裡的意思是做中學一定是最快的～以下為建議學弟妹如果對 UI/UX 有興趣，學校剛好無法滿足你，你可以參考的路徑～～～1.找 Google 大神當時同輩多屬於酒肉朋友（誤），加上一股很不想要麻煩人、自己太菜，所以資源上很難找到 UI 相關的前輩詢問，因此很直覺拜 Google 大神為師。網路上不乏各式各樣新手入門文章，其中我最推的入門文章就是直接先讀蘋果跟安卓的「 Design guideline 」，因為進入這世界要先懂的他使用的語言。如果你跟我一樣是純新手，那讀完真的是疑惑滿臉，這時候可以先怪罪自己英文不夠好，接著會查到早有其他大神翻成中文版了～～～滿心期待的看完中文版還是有很大的機會不知道在幹嘛，但沒關係！因為這時候會有滿滿的求！知！慾！促使自己輸入關鍵字大看ㄧ波～～～我覺得就是用力讀，看不懂的字再繼續查繼續讀，不停的 Loop下去。因為太多人寫自學 UI 相關的文章，而且寫的都很好～所以我只強調讀 Guideline 的重要性，剩下的我想補充一些自己喜歡的網站，以對岸居多，最主要原因是文章更新速度很快，教程資源也多。Ａ.圖文綜合類的 UI 中國類似這樣中國網站很多（優閣網、站酷等），我最常看的是 UI 中國，覺得裡面的討論風氣很好，而且很多大神會分享他們設計執行過程，或是大公司大產品的分析，加上裡面不乏有趣的教程讓我練功，做些小作業會很有成就感。B.文章為主的 優設因為介面設計不單純是視覺的呈現，要追求美以外的東西，所以閱讀文章掌握設計方法，保有使用者為主的設計思維很重要。我也很喜歡看 設計大舌頭、阿里媽媽 梳理自己在執行 UI 的邏輯。如果不只局限於 UI ，人人都是產品經理也很有趣，時不時會挑熱門文章看，個人最喜歡產品需求文檔型的，這種設計過程看了真的很過癮，但剛開始啃的時候挺硬的（現在看也是挺硬的…），建議有實作介面的經驗以後來看，然後許願可以遇到強大 PM 甚至是自己有天成為 PM！C.影音為主的 美啊裡面有各種免費課程、設計論壇的演講（門票很貴的那種）的視頻，有時候我會把他當 Radio 聽，有些設計方法我文章看不懂的時候會看看有沒有相關影片，雖然會遇到專有名詞轉不過來的問題，但加減聽久了就習慣了！我永遠都記得我第一部看的是 IXDC2015 史玉洁的 設計師那點事兒 ，她說對設計要「不忘初心，方得始终」，本來想推給大家，但現在影片竟然要錢了QQ，只好請有興趣的朋友看文字稿了。2.找對的工具使用 Windows 就先不提了，但用 Mac 的學弟妹真心推薦 Sketch，如果狠不下心花那個錢，那不管是用 Ai 還是 Ps 我都覺得……還是真心推薦 Sketch，入門門檻低，真心不騙，物超所值，而且外掛與教學影片甚多，現在業界也都要求 Sketch，所以老師沒有要求沒有教～也請要求自己使用。（Zeplin也是必搭神器）3.找項目開始執行做就對了！！！！！！！但往往不知道從何下手; _ ;所以我建議學弟妹們可以找一個你有興趣的題材，看前期你要做足多少功課競品分析、使用者調查、User story、Flow Chart 或甚至你什麼都不要，直接開畫，我都覺得很 ok（雖然這樣你不會畫得比較好比較快）但能在功課以外時間，出作業給自己就很偉大了，如果還是沒方向，我甚至建議可以直接截圖幾頁喜歡的App，用臨摹去認識每個元件，總之藉由實作來學一定最快～或是找個人出目標給你，例如實習，要是更大膽ㄧ點，就直接出去雷人接案吧！（謝謝當時陪我一同雷人接案的ROSA）4.找實習有些比較大的公司（補習班）像是 YAHOO, KKBOX 都有開出實習計畫，但申請上的前提比較嚴格，你必須有 UI 概念與美美的作品集，在校成績也不能太差。如果進不了大公司，可以選擇以 APP / WEB 開發為主的新創，這些公司也會招收設計實習生，申請條件相較起來比較不會那麼嚴格，對 UI 有概念與有誠意的作品集即可，但要有心理包山包海的設計範圍，畢竟新創的設計師人比較少，沒有大公司來的分工明確，所以有機會學到介面設計，同時也會執行各式設計打雜～所以在學生證還沒被蓋離校之前，找實習機會讓人帶領你各種做中學也是很好的方式！5.找學長姐前輩大大們當然因為自己還很菜，但真的有任何對於介面相關的問題，我都很願意分享。也相信學弟妹不會像當時我一樣沒朋友，身邊多少有介面設計領域的學長姐，其實只要有禮貌敢密敢問，大家ㄧ定都不會吝嗇的！如果有時間更可以參加 UI 聚會或是像 UIUX CAFE 的好 UI 讀書會或是 UI/UX Taipei 的每月講座！這都是很好認識前輩與學習的機會！做個小結論，學校教的科目沒興趣沒關係，至少要請有意識的嘗試並尋找方向做中學吧！嗚～學海無涯，一起繼續學習！From a quick cheer to a standing ovation, clap to show how much you enjoyed this story.小小 UIUX 設計師的心得分享(誤人子弟的那種)AAPD專注於分享數位產品設計的相關資訊，並且致力在平台上創造更多的交流與互動，我們關注UI設計、UX設計、設計師的個人成長、設計趨勢與產業動態等，希望透過這些知識的傳遞，能夠降低每位設計師成長的過程中所遇到的阻礙。歡迎來信投稿：aapdgo@gmail.com

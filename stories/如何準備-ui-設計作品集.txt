應徵 UI 設計工作時，設計師需要透過作品集展示自己的專業能力，替自己爭取面試機會。作品集本身也是一套有使用者、有情境的產品，審查委員看完作品集後的主觀體驗，是決定應徵者是否能晉級、錄用的重要因素，直接影響您的求職機會。因此如何打造出良好的「作品集體驗」，對應徵者來說非常重要。在放作品之前，請先花一兩頁篇幅做自我簡介。審查委員需要知道這份作品屬於誰、想要應徵的職位，以及如何聯繫。這個環節很重要，卻常被應徵者遺忘。直接貼履歷表或專門設計都可以，但要注意：不要把自我介紹拆到作品集的外面，變成兩個檔案。根據統計，有 87% 的企業人資會把這兩個檔案搞散，導致審查委員只拿到作者不詳的作品集。放進作品集的作品數量求精不求多，精選 3–5 個跟應徵職位相關的作品，對整天都在開會的審查委員來說恰到好處。他們可能會用零碎時間（開會、吃飯、等電梯…）來抽空觀看您的作品，因此越具代表性的作品要放越前面。優先選擇由自己主導的設計，或是有知名度的大型專案。作品時間不要超過三年，一來沒辦法反映目前的實力，二來可能會與最新流行風格脫節，顯得過氣。選用適當的作品展示手法，能讓自己的作品大幅加分。動態影片的效果非常好，很吸睛，也能充分展現自己的動態設計能力。可惜很少人會在作品集內放影片。網路上有很多免費的第三方資源，比方說 Wix 的 template 或 mockups，都能讓作品集錦上添花。但這跟套濾鏡修圖一樣，不要濫用這些不屬於您的設計，不要試圖用超級華麗的 template 或誇張透視角的 mockup 掩蓋個人作品的不足。這些小技巧或許可以一時蒙混過關，但絕對逃不了面試官對質逼供，最後只會浪費您的時間。作品除了 mockup、flow，最好加上文字簡介，避免審查委員進行不必要的通靈。文字內容應該包含：作品集最後，別忘了放上行為召喚 (call-to-action)，引導審查委員進行接下來的動作。例如：想看更多作品嗎？這是我的 Behance 連結；想要聯繫我嗎？這是我的 email… 等等。請考慮審查委員可能會在這些場景觀看您的作品集：作品集的載體格式，我推薦最老派的 PDF 檔案。它非常可靠，絕不會出包，在每種設備上都能完美呈現，在公司內部轉發、傳閱很方便。檔案大小盡量控制在 20MB 內，影片或 prototype 可用 hyperlink 方式外連雲端。記得檢查 JPG 壓縮後的品質，尤其是 UI 視覺作品。至於莫名奇妙受設計科系學生喜愛的 Issuu，完全不適合用來當放 UI 作品。Issuu 在許多主流瀏覽器上 render 的效果並不好看，畫面被壓縮得模糊不清，交互也很差勁。刻意模擬紙本書籍翻頁的視覺效果，對 UI 作品集來說不但多餘，還很惱人。它有時還會在您的作品旁邊放廣告，讓審查委員看了一肚子火。但再怎麼糟糕，都遠遠比不上把自己的作品一股腦丟上 Google Drive，然後只甩出一條 shared forder 連結的人。讓審查委員自己想辦法在資料夾裡挖寶，是最糟的「作品集體驗」。當您通過了書面審查階段，被邀請參加面試時，通常要向面試官介紹、解說自己的作品。這時您需要準備一份「面試現場專用作品集」。面試用的作品集內容，可以跟書面審查的版本一模一樣。最好能額外將作品的 prototype 或實際上線產品，下載到手機或電腦裡，供面試者試用，凸顯您的專業溝通能力（這也是面試評估的一環）。要注意的是，許多企業的聯外網路並不太通暢，甚至有些公司怕員工上班看影片，連 YouTube、Google Drive 都開不了，因此務必針對所有雲端作品準備一套 offline 版本。最後用自己的筆電、平板、手機螢幕檢查一遍，確保作品能完美呈現（顏色有跑掉嗎？版本對嗎？字型有缺嗎？）。赴試前挑選 2～3 個最具代表性的作品，針對每個作品準備一段解說，內容應涵蓋：最後祝各位求職順利，找到自己滿意的好工作。若覺得本文對您有幫助，麻煩在下面拍個手兒。From a quick cheer to a standing ovation, clap to show how much you enjoyed this story.資深 UI/UX 設計師，現任職於台灣雪豹科技｜moxamax@gmail.comAAPD專注於分享數位產品設計的相關資訊，並且致力在平台上創造更多的交流與互動，我們關注UI設計、UX設計、設計師的個人成長、設計趨勢與產業動態等，希望透過這些知識的傳遞，能夠降低每位設計師成長的過程中所遇到的阻礙。歡迎來信投稿：aapdgo@gmail.com

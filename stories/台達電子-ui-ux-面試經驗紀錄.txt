嗨，我是 Kris，簡單介紹一下自己，我在大學念產品設計畢業後接著繼續念碩士班，大部分的 UI /UX 實際學習都是在碩士兩年深入研究的，比較特別的是自身跟著指導教授參與過許多產學合作，有實際地與業界開過會，有和PM、R&D 與 Stakeholders 溝通交流的經驗，因此有較為貼近產業且設計流程完整的作品集，我發現大多剛畢業的學生想進入 UI / UX 領域，普遍會有作品集不夠，或者大多作品都只是處於 Concept 的問題，所以這可以算是我的個人優勢。但此篇文章不琢磨在作品集的準備，相關的文章有很多了，此篇著重在「面試經驗談」為主，我覺得台灣在這部分相關的資源還是偏少的，自己面試前受 Google 大神、Ptt 與 Medium 中的資源受益，因此也把自己的經驗分享出來。我想台達電在台灣算有知名度的企業，但台達有在做 UI ? 我想這是許多人不曉得的，我這次應徵的單位是台達電裡的台達研究院，而不是一般的廠區， 「台達研究院」成立於 2013 年，致力於開發與整合大數據分析及物聯網應用與解決方案，以利加速企業轉型，據點分布全球，台灣的部門分別位於台北跟台南。目前我在服義務役當中，因為在服役期間的榮譽假時間只足夠應徵一間公司，所以我先將公司分成以下三類以利篩選：其中 C 是我優先程度較低的考量，前言說過，我碩士兩年其間就是處於這種接案的腳色，優點是在短期間內可以大幅磨練基本功，可惜的就是較少能參與完整的設計開發流程，常常到設計階段的時候，多數功能客戶已經確定，也無法執行後續的驗證階段。當然也不是說所有的 C 類公司都像上述所說，但我還是優先考量 A 與 B 類公司，加上我個人喜歡做創新勝於維護的職務，所以就往公司內部有研發 / 研究單位的公司，最後看到了台達的 JD 幾乎符合我的需求 ! 在此之前我也不知道台達有在應徵 UI / UX 這件事，另一方面這樣的傳統硬體大廠，也願意敞開雙臂去做轉型與創新這件事，我也是非常樂見的，評估完後就決定把這第一次給台達了。我是直接從 104 投遞履歷與作品集連結，兩天後就收到對方 HR 的來電進行面試邀約（覺得好有效率阿），台達電的面試流程說長也不長，說短也不短，以下詳述整個面試流程:要進去台達時先在守衛室換臨時證進入大廳，跟櫃台說一下自己的名字並告知是前來面試的，這時會有另一位 HR 出來帶，第一關算是先考筆試，英文類似多益的考法，單字、文法、克漏字和閱讀，智力測驗偏圖形的判別，邏輯測驗則是考推理為主，給你一些已知條件推敲出題目的答案，時間到了 HR 就會近來把考卷收走，並且讓你稍作等待。一共兩位面試官進來一起面試，一開始請我自我介紹，簡單的帶過自己為何會踏入這個領域，並且闡述過去的專案經驗，說明為何適合這個職位，之後介紹作品，我有額外準備一份面試用的作品簡報，所以主要用投影片來帶過我執行過的專案，兩位面試官分別會從過程問他們想知道的，例如介面顏色選擇的原因、專案中遇到的困難、如何掌握完成專案的時程…等，過程有問到用服務設計來解釋某個誇裝置設計專案的整個使用者旅程（User Journey），這題一方面可能當時有點太緊張，一方面當時有些生疏於服務設計的執行流程，只有碩一曾經修過一門課才初步了解到服務設計這個詞，我自認沒答的很完善。接著從碩士論文的部分提問，包含為何選擇這個主題、文獻放了什麼、前期訪談人數、過程遇到問題是如何克服的、如何篩選受測者…等，最後有聊到我的測試原型是我自學一年左右的程式語言，自己 Coding 實現出來的，這點最後好像給面試官加很多分！最後是由面試官跟我說明目前團隊的組成，以及可能會觸及的專案內容，團隊的工作文化，也提到團隊是以從下到上一起去做重要決策的，團隊組織可以說是較為寬而非深。面試官姊姊在最後離開時，跟我說我的履歷她就收下了，當下真的覺得很感動啊～履歷真的要好好做！兩位面試官離開後（原本以為結束了），請我繼續稍等 HR 進來跟我面試，HR 姊姊這關比較輕鬆，簡單問一下履歷上的問題，以及何時退伍、有沒有投遞其他公司、影響我來台達電工作意願的因素…等，其他也介紹公司福利與薪資結構，主要還說了我通過了一面，接下來還會有第二次面試邀約，詢問我比較可以的時間，之後大部分就比較偏閒聊了～莫約過了半個月後，再次進入台達，這次是三位面試官一起面試，其中一位主管是遠端面試，使用現場的設備進行即時語音通話，一樣是先自我介紹接著作品介紹，二面的面試官專業背景與一面不同，所以提問的方向也不太一樣，較少直接問作品裡的東西，主要問到未來幾年內的職涯規劃、對於設計思考（Design Thinking）與敏捷開發（Agile development）的看法、如何同時處理手上多項的專案、如何與 RD 和 PM 合作工作、設計提案被打槍時如何處理…等。將這題特別獨立拉出來聊一下，這是兩關的面試官都有共同問到的一個問題，對於有經驗的設計師應該不難回答，可以感覺到這是個陷阱題，但對於非本科或是想轉職到UI領域的人可以參考一下我自己認為美感當然對於設計師是非常重要的，甚至是必要的，但 UI 設計跟平面設計或動畫設計有非常大的不同，就是如果一味追求華麗的轉場、酷炫的效果，但操作經驗卻不佳，就不算是好的 UI，更重要的是整個設計呈現的邏輯是否正確，每個 UI Component 的存在與尺寸都是有意義的，當所有元素的運用都是適切的（元素可能包含視覺結構、配色、間距…等），這時畫面自然會有一定的 Balance，儘管我們感受到 UI 的美時不一定會一眼認出，但它可以引導使用者朝向成功的效果。另外一些基本的視覺設計原則可以參考這篇文章後續就是 HR 通知我拿到了 Offer，整個面試過程大概是一個半月，非常開心在即將退伍前可以成功踏入自己熱愛的領域，而整個面試下來對台達這團隊的氣氛感覺很好，從原本只是想說就先試試看，到中間渴望可以進去這間公司，面試後也瞭解台達研究院偏向做一些創新孵化的專案，因研發據點遍佈全球，也有許多到不同地區出差的機會，我覺得是很具有挑戰性的。同時這個職責也比較開放，UX、UI 與 GUI 可能都會碰到，我認為也是好的，不要讓自己侷限在單一的職務裡，常告訴自己不僅是要作為一名 UI / UX designer更要成為一名 Professional designer。最後，拿到了 Offer 僅代表自己通過了最低門檻，之後才是真正的開始，希望此篇文章對於未來想進入這領域的 Designer 們可以作為參考。也謝謝一路上幫助過我的人，謝謝你們。Respect all, Fear none .感謝您的閱讀，對於文章有任何建議與回饋，歡迎在底下留言給我！也可以直接透過我的 Facebook 聯繫我：）From a quick cheer to a standing ovation, clap to show how much you enjoyed this story.UI UX Designer @ Delta Electronics.
